* What Works

  - =bannaInfo.cs= — lists all commands, runs examples.
  - =bannaForEtcHosts= — prints sorted =127.0.22.1xx  <name>.here= lines to stdout,
    one per entry in =tcpPortsAssignedList=, starting at =127.0.22.100=.
  - =updateEtcHosts <filePath>= — invokes =py-dblock.cs -i updateDblocks <filePath>=,
    expanding =bx:dblock:global:run-result-stdout= blocks in that file.
  - =py3/test/verify.sh= runs both commands without error.

* What Is Untested

  - =updateEtcHosts= on the real =/etc/hosts= (requires root; tested only on the
    test fixture =py3/test/hosts=).
  - Full end-to-end with =capDns_update= calling =updateEtcHosts= (capDns stubs not
    yet wired).

* Key Design Decisions

  - =bannaForEtcHosts= outputs to stdout only (via =print()=); no opResults. This
    is intentional — the pyDblock =run-result-stdout= handler captures stdout to
    fill the dblock body.
  - Each output line format is =IP  name.here  # port=NNNNN= so the hosts file
    is self-documenting about which service owns each loopback address.
  - IP assignment starts at =127.0.22.100= and increments sequentially over the
    alphabetically sorted service name list.
  - =updateEtcHosts= delegates entirely to =py-dblock.cs= rather than importing
    pyDblock directly, keeping banna free of a hard dependency on bisos.pyDblock.
